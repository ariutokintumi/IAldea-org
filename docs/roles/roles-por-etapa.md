# Roles por etapa (derivado de la matriz)

Este documento **no sustituye** el CSV; resume la columna narrativa por rol en cada etapa del ciclo de coordinación.

- **Export completo (hoja Role Model + Permission Matrix):** [role-model-permission-matrix-full.csv](role-model-permission-matrix-full.csv)
- **Matriz canónica reducida (repo):** [permission-matrix.csv](permission-matrix.csv)

## Glosario de columnas (fila guía del export)

| Columna | Descripción (export) |
|---------|----------------------|
| **Etapa** | Nombre de la etapa del ciclo de coordinación |
| **Descripción de la etapa** | Cuál es el propósito de esta etapa |
| **Coordination Object** | Palabra clave para detonar el proceso |
| **Qué hace la IA** | Cuál es el rol del agente AI |
| **Capability principal** | Lo que entiende el backend. |
| **Firewall** | Cuáles son las red flags que requieren activar el firewall |
| **Legitimacy trails** | Qué datos se registran onchain para ser verificables |
| **Source** | De quién viene la información |
| **Privacy** | Para quién es visible el output |
| **Review Cycle** | Cada cuándo hay que repetir esta etapa |
| **Secretaría** | Quien lleva la minuta de la reunión |
| **Coordinación** | Quien vela por los objetivos del comité |
| **Miembro de Comité** | Quien participa en la toma de decisiones |
| **Tesorería** | Quien gestiona la tesorería |
| **Validador** | Quien revisa que los compromisos se cumplieron |
| **Ciudadano** | Quien participa en la comunidad |
| **Financiador** | Quien aporta recursos para atender las necesidades de los ciudadanos |
| **Preguntas clave para uX uI** | Preguntas clave para UX/UI |
| **Respuestas** | Respuesta por pregunta (en el export actual suele ir vacía hasta completar la hoja) |

## Ciclo: responsabilidades por rol

### Entender

*Identificar necesidades, tensiones o problemas de la comunidad.*

- **Objeto de coordinación:** Tensión / Necesidad

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra necesidades y antecedentes |
| Coordinación | Organiza el proceso de escucha |
| Miembro de Comité | Ayuda a interpretar el problema |
| Tesorería | Estima capacidad financiera preliminar |
| Validador | No aplica |
| Ciudadano | Da input sobre necesidades |
| Financiador | Puede señalar prioridades financiables sin condicionar decisiones |

### Proponer

*Generar posibles soluciones o caminos de acción.*

- **Objeto de coordinación:** Propuesta

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra propuestas |
| Coordinación | Revisa alineación con objetivos |
| Miembro de Comité | Genera o mejora propuestas |
| Tesorería | Revisa viabilidad presupuestal |
| Validador | No aplica |
| Ciudadano | Puede comentar si el proceso lo permite |
| Financiador | Puede comentar restricciones de financiamiento sin imponer |

### Decidir

*Elegir una propuesta o dejar constancia de un acuerdo.*

- **Objeto de coordinación:** Decisión / Acuerdo

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra la decisión y su razón |
| Coordinación | Verifica que el proceso fue adecuado |
| Miembro de Comité | Participa en la decisión |
| Tesorería | Libera fondos solo si existe autorización humana |
| Validador | No aplica |
| Ciudadano | Consulta resultado cuando sea público |
| Financiador | Consulta resultado y condiciones públicas |

### Ejecutar

*Dar seguimiento a compromisos asumidos.*

- **Objeto de coordinación:** Responsabilidad / Compromiso

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra avances |
| Coordinación | Da seguimiento a responsables |
| Miembro de Comité | Ejecuta actividades |
| Tesorería | Registra uso de recursos |
| Validador | No aplica |
| Ciudadano | Consulta estado público |
| Financiador | Consulta estado agregado |

### Verificar

*Revisar si los compromisos se cumplieron.*

- **Objeto de coordinación:** Rendición de cuentas

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra resultado |
| Coordinación | Coordina revisión |
| Miembro de Comité | Presenta evidencia |
| Tesorería | Reporta uso de recursos |
| Validador | Revisa cumplimiento |
| Ciudadano | Consulta resultado público |
| Financiador | Consulta métricas agregadas |

### Informar

*Comunicar resultados, decisiones o avances.*

- **Objeto de coordinación:** Noticia / Comunicado

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Redacta minuta o comunicado |
| Coordinación | Autoriza mensaje |
| Miembro de Comité | Revisa contexto |
| Tesorería | Revisa datos financieros |
| Validador | Valida exactitud si aplica |
| Ciudadano | Recibe y consulta información |
| Financiador | Recibe reporte agregado |

### Recordar

*Recuperar memoria colectiva para responder preguntas.*

- **Objeto de coordinación:** Memoria

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Consulta historial |
| Coordinación | Consulta historial |
| Miembro de Comité | Consulta historial |
| Tesorería | Consulta historial financiero autorizado |
| Validador | Consulta historial de validaciones |
| Ciudadano | Pregunta sobre memoria colectiva |
| Financiador | Ve métricas agregadas públicas |

### Aprender

*Ajustar acuerdos o prácticas a partir de experiencia.*

- **Objeto de coordinación:** Retroalimentación / Cambio de política

| Rol | Qué hace en esta etapa (matriz) |
|-----|--------------------------------|
| Secretaría | Registra cambios |
| Coordinación | Coordina revisión |
| Miembro de Comité | Propone mejoras |
| Tesorería | Ajusta criterios financieros |
| Validador | Valida impacto |
| Ciudadano | Da feedback |
| Financiador | Da feedback sobre resultados agregados |
